## Key Findings Summary

### Heapsort

- **Time complexity:** Θ(n log n) in worst, average, and best cases — the only O(n log n) comparison sort that is also in-place (O(1) auxiliary space).
- **Insensitive to input order:** Empirically confirmed — times across sorted, reverse-sorted, random, and nearly-sorted inputs at n = 30,000 varied by less than 12%.
- **Cache locality weakness:** Heapsort's non-sequential memory access pattern makes it ~1.8× slower than randomised Quicksort on typical hardware despite equal asymptotic complexity.

### Quicksort vs Heapsort vs Merge Sort

| Distribution | Winner | Note |
|---|---|---|
| Random | Quicksort (1.8× faster than Heapsort) | Best cache locality |
| Sorted | Quicksort | Random pivot prevents O(n²) |
| Reverse sorted | Quicksort / Merge Sort | ~1.6× faster than Heapsort |
| Nearly sorted | Quicksort | 2× faster than Heapsort |
| **Many duplicates** | **Heapsort / Merge Sort** | **Quicksort is 2× slower!** |

The most important result: **two-way-partition Quicksort degrades significantly on inputs with many repeated values** (approaching O(n²)), while Heapsort and Merge Sort remain stable at O(n log n).

### Priority Queue

| Operation | Complexity |
|---|---|
| `insert` | O(log n) |
| `extract_max` | O(log n) |
| `increase_key` | O(log n) |
| `decrease_key` | O(log n) |
| `peek` | O(1) |
| `is_empty` | O(1) |

The auxiliary `{task_id → index}` dictionary is the key to achieving O(log n) for key-change operations (vs O(n) without it).

## Design Highlights

- **Iterative `heapify`** avoids Python's recursion limit and eliminates O(log n) call-stack overhead.
- **Bottom-up `build_max_heap`** constructs the heap in O(n), not O(n log n).
- **`_swap` keeps the index dict consistent** atomically before each array swap — no stale lookups possible.
- **`MinHeapPriorityQueue` via priority negation** — full min-heap functionality with zero code duplication.
- **`TaskScheduler.update_priority`** calls `increase_key` or `decrease_key` automatically based on the direction of change.
